Variable

Unit 1, Primitives

Named location in computer memory. Stores data for use in application. Java is statically typed, meaning that variables must be declared, with a data type, before they are used.

Data Type

Unit 1, Primitives

A descriptive term, indicating what can be done with data, and how much space is required to store it. For instance int variables can be divided, added, multiplied. Basic data types in AP A are int, boolean and double.

Primitive Data Type

Unit 1, Primitives

A data type of known storage requirement, like int, boolean, double.

Reference

Unit 1, Primitives

Think of a reference as the starting memory location for storing data of an unpredicted size

boolean

Unit 1, Primitives

Primitive Data type having only values of either true or false. Named for Boolean logic.

int

Unit 1, Primitives

Primitive numeric data Data type. Occupies 32 bits and can store numbers from -231 to 231 - 1. Whole numbers only.

double

Unit 1, Primitives

Primitive numeric data Data type. Occupies 64 bits and can store numbers from -263 to 264 - 1. Fractional parts included.

Parameter

Unit 1, Primitives

Data that is passed to a method. Formal parameters are described in the defintion of a method, whereas actual parameters are passed to the method in calling code.
Below, rate and balance are formal parameters,

public double interestCalculated(double rate, double balance) { ... }


... and the actual parameters are seen in the call,

double interest = interestCalculated(0.03, 12000);

addition +

multiplication *
division /
modulo %

assignment =

Constant

Unit 1, Primitives

A variable, value of which cannot be changed after initialization. Commonly represented in uppercase. Indicated with keyword final.
final int GOAL_HEIGHT = 8;

Class

Unit 1, Primitives

A template for creating an object or instance.

Object

Unit 1, Primitives

A data model created from a class.

Casting

Unit 1, Primitives

A way to 'temporarily convert' or represent data as another type. For example,
double height = 34.5;
int val = (int) height; // This is a narrowing conversion
int width = 5;
double side = (double)width; // This is a widening conversion

Note, when an int is typecast to a double, information is not lost. However, when a double is typecast to an int, the fractional part is truncated.

OOP

Unit 2, Using Objects

Object Oriented Programming. A software engineering paradigm (technique) in which templates called classes are used to create data models called objects. The application is built around the objects.

Class

Unit 2, Using Objects

A software template from which data models called objects are created.

Object, Instance

Unit 2, Using Objects

A data model created from a software template called a class. An object (also called instance) is 'instantiated' or 'newed up' from a class.

Attribute

Unit 2, Using Objects

A characteristic of an object. Other terms used include instance variable, property, data member, instance data.

Method

Unit 2, Using Objects

A block of code, given a label, and bound to an object (instance method), or class (class method). An instance method is described as something the object can do.

Constructor

Unit 2, Using Objects

A special method that is responsible for instantiating or 'newing up' an object from a class.

Getter or Accessor

Unit 2, Using Objects

A special method that returns the value of a private attribute.

Setter or Mutator

Unit 2, Using Objects

A special method that sets the value of a private attribute.

Overloaded Method

Unit 2, Using Objects

A redeclaration of an existing method, with a different parameter list.

Accessibility Modifier

Unit 2, Using Objects

This governs the visibility of a class, a method, an attribute. private gives access only within the class, whereas public gives access outside of the class. There are other accessibility modifiers.

String

Unit 2, Using Objects

String is a special class that Java uses to represent String instances - strings are a bunch of characters, like "Robbie Roe". String is a reference type, used to create an instance. String instances can be created using the new operator,

String str1 = new String("Have a great day!");

... and by using String literals, String str2 = "Have a great day!";

The String methods you need to know about are: length(), indexOf(..), substring(..), equals(..), compareTo(..)

String instances are immutable, you cannot change them. When you do 'change' an existing String instance, a new String instance is created.

Keyword, static

Unit 2, Using Objects

This keyword indicates that an attribute or a method s bound to a class, not an object. As such, no instantiation is required to access the static quantity. For example, 

        public class SoccerPlayer {
            
...

            static int numberOfPlayers;
        }
    
    In the client code, 

        int playersTotal = SoccerPlayer.numberOfPlayers;

Iteration

Unit 4, Iteration

An action that is repetetive, happens in a loop.

Loop

Unit 4, Iteration

Term given to a construct in computer programming in which a block of code is repeated either a set number of times or infinitely.

While Loop

Unit 4, Iteration

A specific loop that uses the while keyword, a loop variable (sentry variable) and a boolean expression, to govern how many times the loop is executed. Loop runs while the boolean expression (loop condition) is true.

For Loop

Unit 4, Iteration

A specific loop that uses the for keyword, a loop variable or sentry variable and a boolean expression to govern how many times the loop is executed. The for loop construct supports initialization and allows for specification of an action to be taken each time through the loop.

Infinite Loop

Unit 4, Iteration

A loop that that does not have an exit point, the loop condition is always true - this results in a loop that cycles infinitely.

Class

Unit 5, Writing Classes

A software template in Object Oriented Programming. Software models called objects (or instances) are made from a class, in the same way an achitectural blueprint can be used to build multiple houses.

Object

Unit 5, Writing Classes

A software model made from the instructions in a class. Also called an instance. We say the object has been instatiated, or 'newed up' based on the class.

Attribute

Unit 5, Writing Classes

A characteristic or feature of a class or object. Attributes belonging to the instance are also termed

instance variable
data member
instance data
property

By convention, attributes are protected with the private accessibility modifier. Special methods called Getters and Setters get and set attribute values from outside the class.

Method

Unit 5, Writing Classes

This is a block of code tied to an object or a class. It is given a signature describing it's visibility (like public or private), the return type and the name.

Constructor

Unit 5, Writing Classes

A constructor is a special method that has responsibility for instantiating a new object based on the instructions in a class. A constructor is described as an empty constructor if it has no parameters, and a parameterized constructor if it has a parameter list.

Data Structure

Feature of programming languages, supports storing and processing data, showing relationships.

Array

Unit 6, Arrays

Data structure that stores a collection of elements, each identified by an array index.

Array Index, Subscript

Unit 6, Arrays

A whole number valid in range 0 to 1 less than the length of the array. Identifies array element.

Out Of Bounds Error

Unit 6, Arrays

Condition, index less than 0 or greater than or equal to number of array elements.

Array Element

Unit 6, Arrays

Entity (primitive or object) array contains, identified by array index.

Traverse Array

Unit 6, Arrays

Sequencing through the elements of array, typically using a loop.

Enhanced For Loop

Unit 6, Arrays

Loop, sequences through a collection, like an array :
for (Item item : items)
  process(item)

Populate Array

Unit 6, Arrays

Add elements to the array.

Linear Search

Unit 6, Arrays

Search algorithm, sequences through array, comparing each element to target.

Binary Search

Unit 6, Arrays

Search Algorithm, assumes sorted array. Splits array in half repetitively, to find target.

Insert Sort

Unit 6, Arrays

Sorting algorithm, relies on inserting subsequent items into sorted part of list. O(n2), Ω(n).

Select Sort

Unit 6, Arrays

Sorting algorithm, swaps elements to sort list. O(n2), Ω(n2).

O(expression)

Unit 6, Arrays

Big O. Symbol describing worst case scenario in terms of swapping/comparing values in sort process.

Ω(expression)

Unit 6, Arrays

Big Ω. Symbol describing best case scenario in terms of swapping/comparing values in sort process.
